## 🚀 Зачем это нужно

Большинство продуктовых команд не терпят неудачу из-за отсутствия идей.

Они терпят неудачу потому что:

* проверяют гипотезы слишком поздно
* полагаются на интуицию
* смешивают предположения с реальностью
* избегают противоречий

Этот фреймворк помогает ответить на главный вопрос:

> **Стоит ли вообще реализовывать эту идею?**

---

## ⚙️ Как это работает

📊 Диаграмма:
👉 [Обзор архитектуры](./assets/architecture-overview.svg)

Система делит анализ на три независимых слоя:

* **Roles Layer** → внутренняя логика
* **Market Layer** → внешняя реальность
* **Synthesis Layer** → конфликт и вывод

## ⚙️ Фреймворк vs Инструменты

Это не инструмент.

Это способ мышления.

```text
Фреймворк → как думать  
Инструменты → как запускать
```

Можно использовать:

* ChatGPT
* локальные LLM
* API
* IDE

---

## 🔧 Как использовать

1. Сформулировать гипотезу
2. Выбрать релевантные роли
3. Добавить контекст

Далее:

```text
playbooks/run-hypothesis.md
```

---

## 🧬 Принципы

* разделяй внутренние и внешние сигналы
* нет данных → нет утверждения
* противоречия важнее согласия
* решение принимает человек

---

## 🏗 Структура

```text
layers/       → логика анализа  
templates/    → шаблоны запуска  
playbooks/    → сценарии использования  
examples/     → примеры  
architecture/ → устройство системы  
```

---

## 🚀 Что это даёт

* быстрее принимать продуктовые решения
* меньше тратить ресурсы впустую
* прозрачную логику
* меньше когнитивных искажений

---

## ⚠️ Что это НЕ делает

* не генерирует идеи
* не заменяет исследование рынка
* не автоматизирует принятие решений
* не является “магией AI”
