# Persona Builds

Persona build files document how reusable personas were synthesized from interviews and research materials.

Use this directory when a persona is rebuilt from new CustDev evidence.

## Suggested Format

```markdown
---
persona: Application Security Engineer
slug: application-security-engineer
built_on: YYYY-MM-DD
source_interviews:
  - interview-001
  - interview-012
confidence: medium
---

# Persona Build: Application Security Engineer

## Source Set

## Repeated Patterns

## Contradictions

## Changes To Persona

## Evidence Gaps
```

## Rule

A persona build should explain why a persona changed, not only what changed.
